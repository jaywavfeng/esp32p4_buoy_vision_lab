# P4 Buoy Vision 客户需求与验收标准

日期：2026-07-16  
角色：客户验收/测试视角  
范围：ESP32-P4 板端视频采集、推理、存储、USB 导出、Web 调试和三种运行模式。本文只定义客户需求和验收标准，具体实测结果见 `TEST_ISSUES.md`。

## 1. 总体目标

客户需要一套能在板端闭环运行的浮标视觉系统。核心目标不是单独验证某个 demo，而是确认设备在真实使用中可以完成：

- 板端视频采集。
- 板端模型推理。
- 原始视频和推理视频同步存储。
- Web 配置、预览、下载和调试。
- USB U 盘方式批量导出 TF 卡数据。
- 野外无人值守录像时，即使被 USB、reset 或异常中断，也不能丢失已经录到的视频片段。

客户最终可接受的交付形态是：每次录像采集结束后，TF 卡中必须能看到一组可播放、可下载、可追溯的原始视频 `raw_*.avi` 和推理视频 `annotated_*.avi`。

## 2. 三种运行模式

| 模式 | 用户目标 | 必须满足的行为 |
|---|---|---|
| SERVER / Web 服务模式 | 配置、预览、模型验证、录像管理、下载和维护 | Web、API、AP/STA、Ethernet、mDNS 可用；TF 状态可见；页面按钮必须有真实后端效果 |
| FIELD / 野外录像模式 | 无人值守采集、推理、存储 | 采集、推理、raw/annotated 写入并发运行；达到片段长度后自动闭合成对视频；未达到片段长度但被中断时也要整理出成对视频 |
| USB_EXPORT / USB U 盘导出模式 | 离线批量复制 TF 卡数据 | 任意模式下插入 USB 后导出 `P4_BUOY`；Web 明确提示 TF 被 USB 占用；安全弹出、host detach 或物理拔线后自动恢复 Web/SERVER 和 TF 写入能力；一直插着 USB 时可通过 Web 手动恢复/再次导出 |

当前客户指定默认验收配置：

- `recording_segment_ms=1800000`，即 1800 秒一段。
- `field_idle_timeout_ms=30000`，即 30 秒无 Web/stream/download/validate 客户端后自动进入 FIELD。
- 模型默认按当前设备配置使用 Fish31。

## 3. SERVER / Web 服务模式需求

| 功能 | 用户动作 | 验收标准 |
|---|---|---|
| 状态读取 | 打开 Web 首页或请求 `/api/status` | 显示当前模式、网络地址、TF 状态、USB 状态、相机状态、模型状态、录像统计 |
| 保存设置 | 修改片段长度、自动采集倒计时、模型、网络模式后保存 | API 成功返回；页面回显设备实际生效值；重启后配置保持 |
| 立即进入野外录像 | 点击“立即进入野外录像” | 设备真实进入 FIELD；Web 断开前返回 pending；TF 开始写入 raw/annotated |
| 30 秒无连接自动进入 | 打开自动采集后停止访问 Web/stream/download/validate | 30 秒后设备真实进入 FIELD，而不是只显示倒计时 |
| 实时图传 | 点击打开实时图传 | 相机从 standby 唤醒，`/stream` 输出 MJPEG；关闭后可回 standby |
| 模型切换 | 保存 Fish31/TinyCNN/COCO 等模型 | `/api/status` 和后续推理/录像使用保存后的模型，或明确说明固件回退行为 |
| 单图/视频验证 | 运行单图验证或 16 帧视频验证 | 返回任务 ID；状态从 queued/running 进入 done/failed；期间 Web 不应被长请求阻塞 |
| 录像列表 | 查看录像记录 | 一条客户可见记录应同时包含 raw 和 annotated，显示帧数、时长、模型、下载入口 |
| 下载录像 | 点击 raw 或 annotated 下载链接 | 下载到有效 AVI，大小与列表/TF 文件一致 |
| 补帧/重建推理视频 | 对 raw 点击补帧 | 生成或重建对应 annotated，最终帧数与 raw 一致 |
| 清空录像记录 | 点击清空并确认 | 后台清理录像、sidecar、索引和临时文件；完成后列表为空；操作必须有明确风险提示 |
| TF 重试 | 点击检查并重试 TF | 重新挂载/验证 TF；成功后写入验证通过；失败时给出可操作提示 |
| USB 立即导出 | 点击 USB 导出或插入 USB | 停止应用侧 TF 写入，把 TF 导出给电脑，Web 给出 USB 占用状态 |
| USB 恢复存储 | USB 弹出/拔出自动恢复，或一直插着时手动恢复 | TF 交还应用，Web/SERVER 可访问，写读验证通过；失败时 Web 给出可重试提示 |

## 4. FIELD / 野外录像模式需求

FIELD 是客户最关心的核心模式，必须按用户真实场景验收。

| 编号 | 需求 | 验收标准 |
|---|---|---|
| F-01 | 采集、推理、存储并发 | raw 采集持续写入；推理结果同步写入 annotated；慢推理不能阻塞采集 |
| F-02 | 成对视频 | 每个片段必须同时存在 `raw_*.avi` 和 `annotated_*.avi` |
| F-03 | 帧数一致 | 同一片段 raw 和 annotated 帧数一致，时长基本一致 |
| F-04 | 正常片段闭合 | 达到设置片段长度时自动闭合当前一对 AVI，并继续下一对 |
| F-05 | 短录像保留 | 即使只录 10 秒，也必须整理为一对有效 AVI |
| F-06 | reset 中断恢复 | FIELD 录制中 reset 后，启动恢复必须把已写入帧整理为成对 AVI，不能静默丢弃 |
| F-07 | USB 中断恢复 | FIELD 录制中插入 USB 前，当前 raw/annotated 必须先闭合成对视频，再导出 |
| F-08 | 列表不污染 | Web 列表不得出现只有 raw、只有 annotated、损坏 AVI 或不可解释残留文件 |
| F-09 | 退出采集后可验证 | 退出 FIELD 后，应能通过 Web 或 USB 看到刚才生成的视频对 |

## 5. USB 导出需求

| 编号 | 场景 | 验收标准 |
|---|---|---|
| U-01 | SERVER 中插入 USB | Windows 出现 `P4_BUOY`，Web 显示 TF 已被 USB 占用 |
| U-02 | FIELD 中插入 USB | 当前录像先闭合成对视频，再导出 TF |
| U-03 | USB 已插着时启动 | 设备不应卡死；应进入明确的 USB_EXPORT 或 SERVER 状态 |
| U-04 | Windows 安全弹出 | 设备应自动排队恢复 TF；若 Windows 仍占用或恢复失败，Web 必须明确提示“安全弹出/拔线/点击恢复”的兜底路径 |
| U-05 | USB 拔出 | 设备自动恢复 SERVER/Web，TF 挂载并完成写读验证；下一次插入仍应自动导出 |
| U-06 | USB 占用时访问 Web | Web 应明确说明 TF 被 USB 占用，不让用户误以为录像丢失 |
| U-07 | U 盘可见性 | 普通 Windows 用户应能在资源管理器中看到盘符或得到明确操作提示 |

## 6. 验收场景矩阵

| 场景 | 操作 | 预期结果 |
|---|---|---|
| 首次上电配置 | 访问 `http://169.254.100.2/` 或 AP/STA 地址，保存 1800 秒片段和 30 秒倒计时 | 配置回显正确，TF 写读验证通过 |
| 手动进入采集 | 点击“立即进入野外录像” | 设备进入 FIELD，开始写 raw/annotated |
| 正常片段闭合 | FIELD 连续运行到片段长度 | 生成一对 raw/annotated AVI，并继续下一片段 |
| reset 中断 | 1800 秒片段配置下，录制 10-30 秒后 reset | 重启后仍能看到这 10-30 秒对应的一对 raw/annotated AVI |
| 自动进入采集 | 无 Web/stream/download/validate 连接 30 秒 | 设备自动进入 FIELD，并开始写 raw/annotated |
| 自动采集中断 | 自动 FIELD 录制 10-30 秒后 reset 或 USB 导出 | 退出采集后能看到成对视频 |
| USB 导出复制 | 在 Windows 打开 `P4_BUOY` | 可复制 `esp32p4/recordings` 中成对 AVI |
| USB 弹出/拔出恢复 | Windows 安全弹出或拔出 USB | 设备自动恢复 SERVER/Web，TF 可再次写入；若一直插着 USB，则 Web 手动恢复也可完成同样状态切换 |
| Web 下载 | 通过录像列表下载 raw/annotated | 下载文件大小与 TF 文件一致，可播放 |
| 补帧 | 对 raw 执行补帧 | annotated 重建完成，帧数与 raw 一致 |
| 清空记录 | 点击清空并确认 | 录像目录和索引清理完成，页面进度明确 |
